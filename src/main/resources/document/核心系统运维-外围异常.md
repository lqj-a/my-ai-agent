#### 1. 外围系统调用核心接口网络连接超时如何排查？
答：优先排查两端网络连通性、端口放行策略、防火墙及安全组规则；检查外围系统出口 IP 是否加入核心白名单；确认核心对应服务实例运行状态、负载压力是否过高；核对接口地址、端口、域名配置是否错误或过期；临时可检测链路延迟与丢包，长期优化外围调用超时参数、错峰调用，保障跨系统网络链路稳定。

#### 2. 外围对接报文加密解密失败、验签异常怎么处理？
答：检查双方加密算法、密钥证书、签名规则是否一致；确认证书有效期、密钥文件是否损坏、配置是否变更；核对报文编码、换行符、特殊字符、报文格式是否符合对接规范；同步更新过期证书与密钥，统一加解密配置版本，对接异常报文单独日志留存，便于问题溯源复盘。

#### 3. 外围同步数据推送失败、数据丢失或推送重复如何解决？
答：核查外围推送地址、消息队列、回调接口状态是否正常；查看推送日志与报错信息，判断是网络中断、接口报错还是数据格式问题；启用消息重试、消息持久化、死信队列机制；外围与核心双方增加数据对账、流水比对，定时校验同步数据完整性，避免单向数据偏差堆积。

#### 4. 第三方外围系统变更后，核心对接服务批量报错怎么办？
答：第一时间拦截外围新增异常流量，临时关闭异常对接接口或做限流降级；对比变更前后报文字段、协议、请求方式差异，快速适配核心解析规则；联系外围厂商回退变更或临时兼容旧版本协议；外围变更需提前联调、灰度验证，禁止无通知直接生产变更，建立外围变更准入管控流程。

#### 5. 外围节点服务心跳断开、离线告警频繁触发如何处置？
答：检查外围节点网络波动、进程挂机、资源占用过高问题；核对心跳上报地址、上报周期、超时阈值配置是否合理；重启异常外围进程，优化节点服务器资源配置，清理磁盘与内存占用；调整心跳检测策略，避免网络抖动误告警，区分真实离线与短暂网络波动，减少无效告警干扰。

#### 6. 外围文件传输对接失败、对账文件无法正常接收怎么排查？
答：检查文件传输服务（SFTP/FTP）连通性、账号密码权限、目录读写权限；核对文件命名规则、文件格式、编码、大小是否符合约定标准；排查传输链路中断、定时任务未触发、文件生成延迟问题；建立文件接收监控、缺失文件告警、自动重传机制，保障跨系统文件对账稳定流转。
