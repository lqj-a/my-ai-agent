#### 1. 银行核心服务器磁盘使用率过高告警如何处理？
答：优先登录告警节点，通过df -h、du -sh快速定位大日志、临时文件、过期备份包；区分数据库日志、应用运行日志、系统临时文件，安全清理过期 7 天以上无用日志与废弃安装包、压缩包；禁止删除数据库正在写入的 redo、clog、核心运行日志；长期需优化日志轮转策略、配置定时自动清理，合理划分挂载目录，提前设置磁盘使用率预警阈值，避免单分区打满引发系统停服。

#### 2. 核心微服务实例端口探测异常、端口不通是什么原因？
答：常见原因包含容器实例假死、进程正常但端口未监听；网络策略、防火墙、安全组拦截探测端口；服务配置中心端口配置下发异常；服务启动加载配置失败未正常绑定端口；集群节点网络转发异常。处理方式：telnet/curl 验证端口连通性，排查本地端口监听状态，核对 Nacos 配置，重建异常容器实例，检查节点网络白名单与访问策略。

#### 3. 核心应用频繁出现 GC 卡顿、服务响应变慢如何排查？
答：首先查看应用 JVM 堆内存使用、GC 次数与耗时，导出线程堆栈、GC 日志分析内存溢出、内存泄漏、死锁问题；检查是否存在大结果集 SQL、未释放连接、批量大数据压测场景；调整 JVM 堆参数、优化长事务与慢查询，限制单次查询返回数据量；增加监控指标，对内存、GC、线程数做实时告警，规避高峰期资源耗尽。

#### 4. 数据库连接池耗尽、提示连接数爆满怎么解决？
答：临时收紧前端并发、暂停批量调度任务，释放无效闲置数据库连接；排查长事务、未关闭数据库连接、慢 SQL 占用连接等问题；调整连接池最大连接数、空闲超时回收配置；优化业务代码确保连接正常关闭，增加慢 SQL 审计与拦截，定期清理无效会话，从源头控制不合理批量查询。

#### 5. 核心定时调度任务执行失败、进程卡死如何处置？
答：先查看调度日志、系统资源负载（CPU / 内存 / 磁盘），判断是否因资源不足导致任务阻塞；检查定时任务冲突、执行时间重叠、依赖前置任务未完成；重启异常调度进程，手动补跑失败批次；优化任务分片与执行频率，错峰排布批量作业，增加任务超时熔断与失败重试机制，完善调度异常告警。

#### 6. 核心节点时间偏移、时钟不同步会造成什么影响？
答：系统时间不一致会导致交易流水时间错乱、分布式事务异常、缓存过期异常、日志链路追踪混乱、证书鉴权失败、定时任务执行异常。需统一配置服务器 NTP 时间同步服务，定期巡检集群所有节点时钟偏差，及时校准时间，禁止手动随意修改生产服务器系统时间。
